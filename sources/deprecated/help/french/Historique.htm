Historique

3.8.9 - 2 aôut 2007

  • Langue portugaise ajoutée. Merci à Marcelo Schneider pour la traduction.
  • Correction interne: les Makefiles sont abondonnés au profit de fichier projet Visual Studio. MSBuild est maintenant utilisé au lieu de GNU Make.
  • Correction interne: les nouvelles versions "sécurisées" des routines de la librairie standard C sont mainteneant utilisées. Ceci protège Cracklock d'une éventuelle attaque dite de "buffer overflow".

3.8.8 - 7 Novembre 2005

  • Correction : l'installation propose de ne pas installer les modules nécessitant des droits administrateur (comme les extensions shell de l'explorer)
  • Correction : les configurations des applications sont stockées au niveau utilisateur au lieu du niveau machine (ceci permet d'utiliser Cracklock sur des ordinateurs publiques)
  • Correction interne: la construction du projet est maintenant géré par un makefile en remplacement des scripts perl. La compilation est faite avec Visual Studio Toolkit 2003.

3.8.7 - 16 Septembre 2005

  • Correction : incompatibilité avec Windows 2000 ou Windows NT (fonction TzSpecificLocalTimeToSystemeTime)

3.8.6 - 17 Juillet 2005

  • Correction : traduction chinoise et espagnole
  • Correction : affichage du titre des pages du controle tab
  • Correction : Traduction Chinoise et Espagnol
  • Correction : taille des titres des onglets
  • Correction : raccourcis clavier dans le gestionnaire
  • Fonctionnalité : glisser-déposer fonctionner avec le gestionnaire
  • Correction : la sélection d'une langue dans le gestionnaire affecte l'extension shell sans besoin de relancer l'explorer
  • Correction : position des contrôles dans les onglets de l'extension shell
